# HLS transcode

DebridNest can expose optional on-demand HLS playlists for torrent files when clients cannot play the source audio (AC3, EAC3, DTS). Direct `/dl/*` downloads are unchanged.

## Requirements

**ffmpeg** and **ffprobe** must be on `PATH`. The default Docker image does not include them; use the `transcode` compose profile or build with `WITH_FFMPEG=1` (see below).

## Enable

| Variable | Default | Description |
|----------|---------|-------------|
| `DEBRIDNEST_TRANSCODE` | `0` | Set to `1` to register `/hls/*` routes |

Mount the transcode routes from your server integration:

```go
_ = transcode.Mount(r, cfg, manager)
```

## URLs

```
GET /hls/{torrentID}/{fileID}/master.m3u8
GET /hls/{torrentID}/{fileID}/index.m3u8
GET /hls/{torrentID}/{fileID}/segNNN.ts
```

- **master.m3u8** — single-variant master playlist
- **index.m3u8** — media playlist (generated by ffmpeg)
- **seg*.ts** — HLS segments

The torrent must be fully downloaded (`status: downloaded`). Partial/in-progress files return `503`.

## Behavior

1. First request for a file runs **ffprobe** on the audio stream.
2. If the codec is AC3, EAC3, DTS, TrueHD, or DTS-HD, **ffmpeg** remuxes video (`-c:v copy`) and transcodes audio to AAC (`-c:a aac -b:a 192k`).
3. Otherwise ffmpeg remuxes with `-c copy` (no transcode).
4. Output is cached under `{DEBRIDNEST_DATA_DIR}/hls/{torrentID}/{fileID}/`.

Segments use 6-second HLS fragments. Transcode runs once per file; later requests serve cached output.

## Docker

Build with ffmpeg:

```bash
docker build --build-arg WITH_FFMPEG=1 -t debridnest:transcode .
```

Or use the compose **transcode** profile (builds with ffmpeg and sets `DEBRIDNEST_TRANSCODE=1`):

```bash
docker compose --profile transcode up -d debridnest-transcode
```

## Client notes

- Safari and many TV apps prefer HLS; use `/hls/.../master.m3u8` instead of direct MKV when audio is incompatible.
- Transcode adds CPU load on first play; plan capacity accordingly.
- For Infuse/Kodi with direct file access, WebDAV or `/dl/*` may still be preferable when the client supports the source codecs.
