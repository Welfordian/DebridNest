# HLS transcode

DebridNest can expose optional on-demand HLS playlists for torrent files when clients cannot play the source audio (AC3, EAC3, DTS). Direct `/dl/*` downloads are unchanged.

## Requirements

**ffmpeg** and **ffprobe** must be on `PATH`. The default Docker image does not include them; use the `transcode` compose profile or build with `WITH_FFMPEG=1` (see below).

## Enable

| Variable | Default | Description |
|----------|---------|-------------|
| `DEBRIDNEST_TRANSCODE` | `0` | Set to `1` to register `/hls/*` routes |

Mount the transcode routes from your server integration:

```go
signer := links.NewSigner(cfg.LinkSecret, cfg.PublicURL, cfg.Host, cfg.LinkTTL)
_ = transcode.Mount(r, cfg, manager, signer)
```

## URLs

HLS URLs are signed with the same `DEBRIDNEST_LINK_SECRET` and `DEBRIDNEST_LINK_TTL_HOURS` model used by direct download links. Use `transcode.MountPath(cfg, torrentID, fileID)` or `links.Signer.SignHLSAsset` to create the master playlist URL. Do not expose raw `/hls/{torrentID}/{fileID}/...` paths.

```
GET /hls/{torrentID}/{fileID}/master.m3u8?expires={unix}&sig={signature}
GET /hls/{torrentID}/{fileID}/index.m3u8?expires={unix}&sig={signature}
GET /hls/{torrentID}/{fileID}/segNNN.ts?expires={unix}&sig={signature}
```

- **master.m3u8** — single-variant master playlist
- **index.m3u8** — media playlist (generated by ffmpeg, served with signed segment URLs)
- **seg*.ts** — HLS segments, subject to `DEBRIDNEST_DOWNLOAD_RATE_LIMIT_MBPS`

The master playlist returns a signed media playlist URL using the same expiration as the master URL. The media playlist rewrites local segment URIs to signed segment URLs. Missing, invalid, or expired signatures return `403`. The torrent must be fully downloaded (`status: downloaded`). Partial/in-progress files return `503`.

## Behavior

1. First request for a file runs **ffprobe** on the audio stream.
2. If the codec is AC3, EAC3, DTS, TrueHD, or DTS-HD, **ffmpeg** remuxes video (`-c:v copy`) and transcodes audio to AAC (`-c:a aac -b:a 192k`).
3. Otherwise ffmpeg remuxes with `-c copy` (no transcode).
4. Output is cached under `{DEBRIDNEST_DATA_DIR}/hls/{torrentID}/{fileID}/`.

Segments use 6-second HLS fragments. Transcode runs once per file; later requests serve cached output.

## Docker

Build with ffmpeg:

```bash
docker build --build-arg WITH_FFMPEG=1 -t debridnest:transcode .
```

Or use the compose **transcode** profile (builds with ffmpeg and sets `DEBRIDNEST_TRANSCODE=1`):

```bash
docker compose --profile transcode up -d debridnest-transcode
```

## Client notes

- Safari and many TV apps prefer HLS; use a signed `/hls/.../master.m3u8` URL instead of direct MKV when audio is incompatible.
- Transcode adds CPU load on first play; plan capacity accordingly.
- For Infuse/Kodi with direct file access, WebDAV or `/dl/*` may still be preferable when the client supports the source codecs.
